# 第一章

  1. LAN（Local Area Networks）局域网

  2. WAN（Wide Area Networks）广域网

  3. ISP，Internet Service Providers：互联网服务提供商 11

  4. ISO，International Organization for Standardization，国际标准化组织

  5. OSI（开放系统互联参考模型，Open Systems Interconnection Model）

    • 是由国际标准化组织（ISO）于 1984 年发布的一个标准化网络通信模型。OSI 模型将网络通信分为七个层次：......1

  6. TCP/IP（传输控制协议 / 互联网协议模型），包含四个层次：....

  7. 通信方式的三种类型：

    • 单工通信（Simplex）：数据只能在一个方向上传输，无法实现双向通信。

    • 半双工通信（Half-Duplex）：数据可以在两个方向上传输，但在某一时刻只能在一个方向上传输。

    • 全双工通信（Full-Duplex）：数据可以在两个方向同时传输，发送和接收可以同时进行。11

  8. PDU（Protocol Data Unit）协议数据单元：协议数据单元，是指在分层网络结构，例如在开放式系统互联（OSI）模型中，在传输系统的每一层都将建立协议数据单元（PDU）。

# 物理层

  1. 无屏蔽双绞线（Unshielded Twisted Pair，UTP）

    • 易于安装且成本较低，线薄接口小

    • 三种类型：

      • 直通线 Straight Cable：连接不同类型的设备

      • 反转线 Rollover Cable：配置路由器 / 交换机

      • 交叉线 Crossover Cable：连接相同的设备

  2. 有屏蔽双绞线 STP

  3. 同轴电缆 Coaxial Cable

  4. Collision Domains 冲突域

    • 指在同一个网络环境中，多个设备在同一时间发送数据时，可能会发生 数据冲突（Collision）的区域

    Broadcast Domain 广播域

    • 指在一个局域网内，通过 广播（Broadcast）方式发送的数据包可以被所有设备接收到的区域

  5. code 码元

    • 在使用时间域的波形表示数字信号时，代表不同离散数值的基本波形

  6. NRZ 不归零编码

  7. RZ 归零编码

  8. 时分复用 TDM（ Time Division Multiplexing）111111

    • 时分复用（Time Division Multiplexing，TDM）是将时间划分为一段段等长的时分复用（TDM）帧， 每个时分复用的用户在每个 TDM 帧中占用固定序号的时隙。

    • STDM： 统计时分复用

  9. 频分复用 FDM（ Frequency Division Multiplexing）

    • 用户在分配到一定的频带后，在通信过程中自始至终都占用这个频带。

  10. 波分复用 WDM（ Wavelength Division Multiplexing）

    • 就是光的频分复用：频率和波长是成倒数关系的。

  11. 码分复用 CDM（Code Division Multiplexing）

    • 常用的名词是码分多址 CDMA (Code Division Multiple Access)

    • 各用户使用经过特殊挑选的不同码型，彼此不会造成干扰。

# 数据链路层

  1. MAC Media Access Control 介质访问控制

    • 介质访问控制是计算机网络中 数据链路层 的一个子层，专门负责管理多个设备如何共享通信介质。

  2. LLC Logical Link Control 逻辑链路控制

    • 逻辑链路控制是 数据链路层（Data Link Layer） 的一部分，用于提供 数据链路层服务，并在不同类型的网络协议之间提供统一的接口。

  3. CSMA/CD Carrier Sense Multiple Access with Collision Detection）带冲突检测的载波侦听多路访问 11111

    • 适配器边发送数据边检测信道上的信号电压的变化情况，以便判断⾃⼰在发送数据时其他站是否也在发送数据。若发现碰撞，⽴即停⽌发送，按退避算法等待⼀段随机时间后再次发送。

  4. CSMA/CA (Carrier Sense Multiple Access with Collision Avoidance) 避免冲突的载波侦听多路访问 1

    • 发送站点在发送数据前，以控制短帧刺激接收站点发送应答短帧，使接收站点周围的站点监听到该帧，从而在一定时间内避免数据发送

  5. NIC Network Interface Controller 网卡

# 网络层

  1. ARP Address Resolution Protocol 地址解析协议 11

    • 使计算机能够查找与 IP 地址关联的计算机的 MAC 地址。

  2. RARP（Reverse Address Resolution Protocol，反向地址解析协议）11111

    • 将 MAC 地址解析为对应的 IP 地址。

  3. IGP （Interior Gateway Protocols）内部网关协议

    • 在自治系统中使用

    • 又分为：

      • 距离矢量协议 DVP（Distance-Vector Protocols）

        • 从邻居的角度查看网络拓扑

      • 链路状态协议 LSP（Link State Protocols）

        • 获取整个网络拓扑的通用视图

  4. EGP（Exterior Gateway Protocols）外部网关协议

    • 用于在自治系统之间路由数据包。

  5. RIP（Routing Information Protocol）路由信息协议

    • 基于距离矢量的内部网关协议

  6. IGRP（Interior Gateway Routing Protocol）内部网关路由协议

    • 基于距离矢量的内部网关协议。**EIGRP（Enhanced IGRP）** 是它的升级版。

  7. OSPF（Open Shortest Path First）最短路径优先协议 1

    • 基于链路状态的内部网关协议，消耗 CPU 和内存

  8. VLSM Variable Length Subnet Mask 可变长度子网掩码 1

    • 允许单个自治系统的网络具有不同的子网掩码

  9. CIDR Classless InterDomain Routing 无类域间路由 11

    • 一种用于 IP 地址分配和路由的机制。核心特点是无类别，采用斜杠表示法，可以聚合路由。

  10. ICMP（Internet Control Message Protocol）因特网控制报文协议 111111

    • 为了提高 IP 数据报交付成功的机会（消息管理和协商），允许主机或路由器报告差错情况和提供有关异常情况的报告

  11. PING（Packet InterNet Groper）数据包因特网探索器

    • 用来测试两个主机之间的连通性

  12. IGMP（Internet Group Management Protocol）互联网组管理协议

    • 专门用于 IPv4 网络 中的组播（Multicast）通信管理

# 传输层

  1. TCP（Transmission Control Protocol）传输控制协议

    • TCP 是一种面向连接的、可靠的传输协议，适用于需要确保数据正确性和顺序的通信场景。

  2. UDP（User Datagram Protocol）用户数据报协议 1

    • UDP 是一种无连接的、快速的传输协议，适用于对速度要求高而对可靠性要求低的场景。

  3. Socket 套接字

    • 是网络通信的一个抽象概念，它是应用层和传输层之间的接口，用于实现网络数据通信。

  4. NAT（Network Address Translation）网络地址转换 1

    • 在 IP 数据包头中将一个地址交换为另一个地址，是网络地址即将用完的解决方案之一

  5. PAT（端口地址转换，Port address translation）

# 应用层

  1. URL（Uniform Resource Locator）统一资源定位符 1

    • 一种标准化的资源标识方法，用于定位互联网上的资源（如网页、文件、图片等）

  2. HTTP（HyperText Transfer Protocol，超文本传输协议）111

    • 一种应用层协议，用于在客户端和服务器之间传输超文本（如 HTML 文档）、图片、视频等资源。TCP，80

  3. HTML（HyperText Markup Language，超文本标记语言）11

    • 用于创建和设计网页的标记语言。

  4. FTP （File Transfer Protocol）文件传输协议 11

    • 一种可靠的，面向连接的传输文件协议，使用 TCP，20/21

  5. TFTP （Trivial File Transfer Protocol）简单文件传输协议

    • 一种简单的文件传输协议，使用 UDP， 69

  6. SMTP（Simple Mail Transfer Protocol，简单邮件传输协议）11

    • 是一种用于电子邮件发送的协议，邮件发送，登录发送等操作。TCP，25

  7. POP3（Post Office Protocol version 3，邮局协议版本 3）

    • 是一种用于接收邮件的协议. TCP，110

  8. MIME（Multipurpose Internet Mail Extensions，多用途互联网邮件扩展）

    • 是一种互联网标准，用于扩展电子邮件协议，使其能够支持文本之外的多种类型的数据

  9. SNMP（Simple Network Management Protocol，简单网络管理协议）

    • 是一种应用程序层协议，可简化网络设备之间的管理信息交换。UDP, 161/162

  10. DNS（Domain Name System）域名系统 1111

    • 是网络上的服务，该服务管理域名并响应客户端将域名转换为关联 IP 地址的请求。UDP，53

  11. DHCP（Dynamic Host Configuration Protocol）动态主机配置协议

    • 是一种网络管理协议，用于自动分配 IP 地址和其他网络参数（如子网掩码、默认网关、DNS 服务器）给网络中的设备。UDP, 67/68

  12. Telnet 协议

    • 用于远程连接主机。TCP, 23

# 路由器

解决路由环路问题的四个办法：

  1. 路由中毒（Route Poisoning）1

    • 路由中毒，信息在路由表中失效的时候，把该表项的的度量值（metric）设为无穷大 16。

  2. 水平分隔（Split Horizon）11

    • 从某个端口收到的报文信息，不能再从该端口发送回去

  3. 定义最大值（Maximum）

    • 设置最大跳数

  4. 计时器（Hold-Down Timers）

    • 标记为不可达时启动一个计时器，若在计时期间收到新的、更优的路径信息，会更新；否则，计时结束还没有新的有效路径信息，则删除该条目

# 路由协议

  1. 指定路由器（Designated Router，DR）：只有在多路复用的链路下使用，被选举出来

  2. 备用指定路由器（Backup Designated Router，BDR）：如果 DR 坏了，再次选举会出现问题，如果 DR 损坏，BDR 立即成为 DR，目的是提高交换效率

  3. ABR（Area Border Router）：区域边界路由器，有一定的约束（一个路由器最多连接 3 个路由器）。

  4. TTL（Time To Live）被路由器丢弃之前允许通过的最大网段数量

# 局域网交换与 VLAN

  1. 对称交换（Symmetric Switching）

    可在具有相同带宽（10/10 Mbps 或 100/100 Mbps）的端口之间提供交换连接。

  2. 非对称交换（Asymmetric Switching）

    通过将带有服务器的网段连接到更高带宽的端口（100 Mbps），减少了服务器上潜在瓶颈的可能性。

  3. 交换机的交换方式：

    1. 储存转发（Store-and-Forward）

      • 交换机接收整个帧，最后校验其 CRC，然后再将其发送到目的地。

    2. 直通（Cut-through）

      • 分为两种。

      • Fast forward switching 快速转发切换：仅在立即转发帧之前检查目标 MAC，即只要看到帧的目的地址就转发，而不看帧的后面的部分（Cut through 有的时候也直接指这个）

      • Fragment Free 碎片释放：在转发帧之前读取前 64 个字节以减少错误：避免碰撞和帧碎片

  4. 生成树协议（STP, The Spanning-Tree Protocol）1

    • 是一种网络协议，用于以太网交换机环境中防止网络中的 环路 问题。

  5. VLAN（Virtual Local Area Network）虚拟局域网

    • 一种在局域网（LAN）中实现逻辑分段的技术，通过将一个物理网络划分为多个虚拟网络，实现对广播域的隔离。

# 广域网

  1. PPP（Point-to-Point Protocol）点对点协议 111111

    • 一种数据链路层通信协议，主要用于在两点之间建立直接连接的链路。

  2. CHAP（Challenge Handshake Authentication Protocol）挑战握手认证协议 111

    • 一种网络认证协议，常用于点对点协议（PPP）连接中，用于验证远程客户端的身份。三次握手，避免了密码的明文传输，更安全

  3. ISDN（Integrated Services Digital Networks）综合数字服务网络

    • 允许通过现有电话线传输数字信号，提供远程站点的连接

  4. ADSL（Asymmetric Digital Subscriber Line）非对称数字用户线路

    • 一种宽带接入技术，通过普通电话线（铜线）提供高速互联网接入

  5. SONET（Synchronous Optical Network）同步光纤网

    • 一种高性能的光纤通信标准

# 网络安全

  1. ACL（Access Control Lists）访问控制列表

    • 是指令列表，它告诉路由器允许或拒绝什么类型的数据包。
